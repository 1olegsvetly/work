1. ОБЩАЯ ИНФОРМАТИКА И ФИЛОСОФИЯ ПРОЕКТА
1.1. Название проекта
Konoden — Сайт по монетизации YouTube и подключению AdSense под ключ для российских авторов.
1.2. Целевая аудитория
Российские YouTube-блогеры с аудиторией от 1000 подписчиков
Авторы каналов, столкнувшиеся с блокировкой российских AdSense-аккаунтов (санкции США с сентября 2024 года)
Владельцы каналов, которые хотят перепривязать AdSense с российского на иностранный
Создатели контента, желающие подключить монетизацию с нуля
Блогеры, которым нужна помощь с налоговой формой W-8BEN и подтверждением адреса
1.3. Ключевая ценность (УТП)
"С нами проще!" — основной слоган и философия проекта. Каждый этап подключения монетизации сопровождается указанием того, как Konoden упрощает этот процесс для клиента. Вместо того чтобы проходить все 7 этапов самостоятельно с риском ошибок, блокировок и потери времени, клиент получает:
Самостоятельно	С Konoden
~300 000 ₽ и выше на ошибках	Фиксированная цена
Риск блокировки канала	Гарантия безопасности
Нет поддержки	Сопровождение 24/7
Месяцы ожидания	7–10 дней
Заполнение форм вслепую	Профессиональная помощь
1.4. Контекст и проблема
С сентября 2024 года США вводят новые санкции в отношении России, в рамках которых американские компании не смогут иметь финансовые отношения с российскими контрагентами. Google не сможет отправлять выплаты российским авторам. Все аккаунты AdSense, открытые в России, будут деактивированы. Решение — перепривязать аккаунт AdSense с российского на иностранный и вывести остаток средств. Konoden предлагает готовые, чистые, официальные аккаунты AdSense, открытые в Армении и других странах.

4. WOW-ЭФФЕКТЫ (9 ДИНАМИЧНЫХ ЭФФЕКТОВ)
4.1. Эффект 1 — "ПАРТИКЛЫ ДОХОДА" (Canvas 2D)
Описание: Мелкие частицы ($, €, ₽, цифры) плавают на фоне hero-секции
Реакция: При скролле частицы реагируют на направление (параллакс)
Цвет: Золотой (#FFD700) с прозрачностью 0.3-0.6
Количество: 30-50 частиц
Скорость: Медленное дрейфование
Технология: Canvas 2D + requestAnimationFrame
4.2. Эффект 2 — "СЧЁТЧИК ЦИФР" (Scroll-triggered)
Описание: При появлении блока статистики цифры анимированно набираются
Примеры: 0 → 100 000+ подписчиков, 0 → 100 000+ выпускников
Длительность: 2.5 секунды
Эффект: ease-out, с запятой-разделителем
Технология: Intersection Observer + requestAnimationFrame
4.3. Эффект 3 — "ГЛИТЧ-ТЕКСТ" (CSS + JS)
Описание: На ключевых заголовках при загрузке страницы
Текст: "МОНЕТИЗАЦИЯ YOUTUBE" мерцает с RGB-разложением
Длительность: 1.5 секунды, затем стабилизируется
Применение: Только на H1 главной страницы
Технология: CSS text-shadow + JS-анимация
4.4. Эффект 4 — "ПРОГРЕСС-БАРЫ УСЛУГ" (Scroll-triggered)
Описание: Горизонтальные полосы заполняются при скролле
Показывают: % готовности этапов (AdSense, W-8BEN, PIN)
Цвет: Зелёный (#00C853) для завершённых, красный (#FF0000) для активных
Анимация: 1.5s ease-out
Технология: Intersection Observer + CSS transitions
4.5. Эффект 5 — "ПЛАВНОЕ ПОЯВЛЕНИЕ КАРТОЧЕК" (Intersection Observer)
Описание: Карточки появляются с translateY(50px) + opacity(0) → translateY(0) + opacity(1)
Задержка: 0.15s между карточками (каскадный эффект)
Длительность: 0.6s
Easing: cubic-bezier(0.25, 0.46, 0.45, 0.94)
4.6. Эффект 6 — "ПУЛЬСИРУЮЩИЕ КНОПКИ CTA" (CSS Animation)
Описание: Кнопки имеют пульсирующее кольцо
Эффект: box-shadow расширяется и исчезает в цикле
Длительность: 2s, infinite
Цвет кольца: rgba(255, 0, 0, 0.3)
4.7. Эффект 7 — "ПАРАЛЛАКС СЛОЁВ" (Scroll-based)
Описание: Фоновые элементы движутся медленнее контента
Скорости: data-speed="0.3", "0.5", "0.8"
Эффект: Создаёт глубину и объём
4.8. Эффект 8 — "ТЕКСТОВАЯ АНИМАЦИЯ ПЕЧАТИ" (Typewriter)
Описание: В блоке "Схема подключения" текст появляется посимвольно
Скорость: 50ms/символ
Технология: JS + CSS cursor animation
4.9. Эффект 9 — "ИНТЕРАКТИВНЫЙ ГРАФИК ДОХОДА" (Canvas/Chart.js)
Описание: Анимированный график роста дохода
Эффект: Линия рисуется при скролле, точки подсвечиваются с tooltip
Цвет: Градиент от зелёного к золотому

12. ИНТЕРАКТИВНЫЕ ИНСТРУКЦИИ (ДОПОЛНИТЕЛЬНЫЙ АКЦЕНТ)
На странице /guides.html, помимо основного роадмапа, мы добавляем интерактивные симуляторы, которые показывают пользователю сложность самостоятельного прохождения этапов и то, как Konoden упрощает процесс.
12.1. Симулятор "Заполнение W-8BEN"
Описание: Показываем форму W-8BEN с полями. Пользователь пытается заполнить её самостоятельно, но при наведении на каждое поле появляется подсказка "С Konoden мы заполняем это за вас". В конце — сравнение: "Вы потратили 15 минут, но допустили 3 ошибки. С Konoden — 0 ошибок за 5 минут."
Техническая реализация: HTML-форма с JS-валидацией, которая показывает ошибки при неправильном заполнении (например, неверный формат TIN).
12.2. Симулятор "Выбор страны"
Описание: Пользователь выбирает страну из списка (Грузия, Казахстан и т.д.). Для каждой страны показываются требования (ВНЖ, банковский счёт, налоги). Затем появляется сравнение "С Konoden мы подберём оптимальную страну без вашего участия".
12.3. Симулятор "Ожидание PIN-кода"
Описание: Таймер, показывающий, сколько дней прошло с момента запроса PIN-кода. Среднее время — 2-4 недели. Появляется сообщение: "Без Konoden вы ждали бы ещё 2 недели. С нами — мы ускоряем процесс через альтернативную верификацию."
12.4. Интерактивный таймлайн "С нами vs без нас"
Визуализация: Две параллельные временные линии:
Верхняя: "Без Konoden" — все этапы растянуты, много красных зон (ошибки, ожидание)
Нижняя: "С Konoden" — этапы сжаты, зелёные зоны (быстро, без ошибок)
Анимация: При скролле линии заполняются, показывая разницу во времени.
Итог: "С Konoden вы экономите в среднем 2 месяца и 300 000 ₽."

13.1.7. Core Web Vitals оптимизация
LCP (Largest Contentful Paint) < 2.5с: Использовать lazy loading для изображений, оптимизировать главное изображение hero, использовать WebP и srcset.
FID (First Input Delay) < 100ms: Defer JavaScript, минимизировать главный поток, использовать async/defer для скриптов.
CLS (Cumulative Layout Shift) < 0.1: Все изображения и видео должны иметь явно заданные размеры width/height, избегать динамических вставок, которые сдвигают контент.
13.1.8. ЧПУ (человеко-понятные URL)
Настроить mod_rewrite в .htaccess для преобразования:
/index.html → /
/services.html → /services
/cases.html → /cases
/guides.html → /guides
/brief.html → /brief
/admin/... → /admin/...
Пример .htaccess:
apache
RewriteEngine On
RewriteRule ^services$ services.html [L]
RewriteRule ^cases$ cases.html [L]
RewriteRule ^guides$ guides.html [L]
RewriteRule ^brief$ brief.html [L]

16.2. Переключение темы (JavaScript)
javascript
// Определяем текущую тему из localStorage
const currentTheme = localStorage.getItem('theme') || 'dark';
document.documentElement.classList.toggle('light-theme', currentTheme === 'light');
// Функция переключения
function toggleTheme() {
  const isLight = document.documentElement.classList.contains('light-theme');
  const newTheme = isLight ? 'dark' : 'light';
  document.documentElement.classList.toggle('light-theme', newTheme === 'light');
  localStorage.setItem('theme', newTheme);
  updateThemeIcon(newTheme);
}
function updateThemeIcon(theme) {
  const icon = document.getElementById('theme-icon');
  if (theme === 'light') {
    icon.textContent = '☀️';
    icon.title = 'Переключить на тёмную тему';
  } else {
    icon.textContent = '🌙';
    icon.title = 'Переключить на светлую тему';
  }
}
// Инициализация
document.addEventListener('DOMContentLoaded', () => {
  updateThemeIcon(currentTheme);
  document.getElementById('theme-toggle').addEventListener('click', toggleTheme);
});

17. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ К СКОРОСТИ И БЕЗОПАСНОСТИ
17.1. Оптимизация скорости загрузки
Минификация CSS/JS: Использовать инструменты (Webpack, Gulp, или просто онлайн-минификаторы) для уменьшения размера файлов.
Gzip сжатие: Настроить на сервере сжатие всех текстовых ресурсов (HTML, CSS, JS, JSON) через gzip или Brotli.
CDN для статики: Подключить Cloudflare или другой CDN для раздачи изображений, CSS, JS.
Кэширование браузера: Установить заголовки Cache-Control для статики (например, на 1 год для изображений, на 1 день для HTML).
Preload критических ресурсов: Добавить <link rel="preload"> для основных шрифтов, CSS, критических изображений.
Font-display: swap: Для шрифтов использовать font-display: swap, чтобы текст отображался сразу, пока шрифт загружается.
Async/Defer для JS: Скрипты, не влияющие на отрисовку, загружать с атрибутами async или defer.
Ленивая загрузка изображений: loading="lazy" для изображений вне видимой области.
Оптимизация шрифтов: Использовать только нужные начертания (Regular, Medium, SemiBold, Bold) и подмножество кириллицы, чтобы уменьшить размер шрифтов.
17.2. Безопасность
HTTPS: Обязательно, с сертификатом Let's Encrypt.
Защита от XSS: Экранировать все пользовательские данные (из JSON при выводе в админке).
Защита от CSRF: В формах использовать CSRF-токены.
Защита от SQL-инъекций: Не используется SQL (используем JSON), но если будет БД, то PDO с подготовленными запросами.
Ограничение доступа к /admin: Защита паролем, сессии, ограничение по IP (опционально).
Защита от брутфорса: После 5 неудачных попыток входа блокировка на 15 минут.
Политика безопасности контента (CSP): Настроить заголовок Content-Security-Policy для ограничения источников скриптов, стилей и т.д.
Защита /data/ папки: Через .htaccess запретить прямой доступ к JSON-файлам: Deny from all.
17.3. Хостинг и серверные требования
PHP: Версия 7.4 или выше (рекомендуется 8.0+).
Веб-сервер: Apache с mod_rewrite (для ЧПУ) или Nginx.
Память: Минимум 512 МБ RAM, рекомендуется 1 ГБ.
Дисковое пространство: Минимум 1 ГБ для изображений и данных.
Поддержка JSON: Встроенная в PHP.
Поддержка Gzip/Brotli: Включена на сервере.
Кэширование на сервере: Можно использовать OPcache для PHP.
