### 1.3 WOW-эффекты (минимум 4, характерные для магазинов аккаунтов)

#### Эффект 1: "Живые счетчики" (Live Counters)
- При загрузке страницы цифры плавно увеличиваются от 0 до актуального значения
- Применение: количество проданных аккаунтов, количество довольных клиентов, uptime сервиса
- **Технология**: Framer Motion + useInView
- **Длительность**: 2 секунды, easing: easeOut

#### Эффект 2: "Пульсирующие карточки" (Pulse Cards)
- Карточки товаров с тегом "🔥 Хит" или "⭐ Новинка" имеют легкую пульсацию glow-эффекта
- При наведении — усиление glow + подъем карточки на 8px (translateY)
- **Технология**: CSS box-shadow animation + Framer Motion hover
- **Цвет glow**: соответствует акцентному цвету темы

#### Эффект 3: "Параллакс-фон" (Parallax Background)
- Фон Hero-секции с легким параллакс-эффектом при скролле
- Декоративные элементы (иконки соцсетей) плавают с разной скоростью
- **Технология**: Framer Motion useScroll + useTransform
- **Скорость**: 0.3x — 0.7x от скорости скролла

#### Эффект 4: "Магнитная кнопка" (Magnetic Button)
- Кнопки "Купить" и CTA при наведении мягко притягиваются к курсору
- Радиус притяжения: 50px
- **Технология**: Framer Motion + mouse position tracking
- **Интенсивность**: 0.3 (мягкое притяжение без агрессии)

#### Эффект 5: "Стеклянный оверлей" (Glassmorphism Modal)
- Всплывающее окно товара с backdrop blur + полупрозрачный фон
- Плавное появление: scale 0.95 → 1, opacity 0 → 1
- **Технология**: CSS backdrop-filter + Framer Motion AnimatePresence

#### Эффект 6: "Бегущая строка" (Marquee Ticker)
- В Top Bar или под Header — бегущая строка с актуальными предложениями
- Бесконечная анимация, плавная, без рывков
- **Технология**: CSS animation marquee

### 5.6 Всплывающее окно товара (Modal) — SEO-оптимизированное

**URL структура (ЧПУ для SEO):**
```
/item/{slug}/

Пример:
/item/fb-accounts-verified-by-email-there-is-no-email-in-the-set-female-the-account-profiles-may-be-empty-or-have-limited-entries-such-as-photos-and-other-information-cookies-are-included-accounts-are-reg/
```

**Техническая реализация:**
- При открытии модального окна: URL обновляется через History API (pushState)
- При прямом переходе по URL: открывается полная страница товара (SSR)
- При закрытии модала: URL возвращается к предыдущему
- **Meta-теги**: Генерируются сервером для каждого товара
- **Schema.org**: Product schema с ценой, наличием, рейтингом

**Содержимое модального окна:**
1. Иконка соцсети (крупная)
2. Название товара (H1)
3. Полное описание
4. Характеристики (таблица)
5. Цена (крупно)
6. Количество в наличии
7. Селектор количества
8. Кнопка "КУПИТЬ" → /oplata/
9. Кнопка "Закрыть"

### 6.3 Переключение темы в админке

**Путь**: Админка → Настройки → Внешний вид → Цветовая схема

| Опция | Описание |
|-------|----------|
| **Тема** | Dark / Light / Auto (по системе) |
| **Акцентный цвет** | Color picker для `--accent-primary` |
| **Фон страницы** | Color picker для `--bg-primary` |
| **Цвет текста** | Color picker для `--text-primary` |
| **Предпросмотр** | Live preview изменений |
| **Применить** | Сохранение в БД, мгновенное применение |

**Техническая реализация:**
- CSS Custom Properties (variables) в :root
- Тема сохраняется в БД (таблица settings)
- При загрузке страницы: CSS-переменные инжектируются в <style> в <head>
- Переключение без перезагрузки (JavaScript)

### 9.2 Процесс импорта

```
1. Пользователь выбирает файл CSV
2. Система парсит заголовки и показывает маппинг
3. Пользователь подтверждает сопоставление колонок
4. Валидация каждой строки
5. Импорт пачками по 100 записей (batch processing)
6. Прогресс-бар с текущим статусом
7. Финальный отчет с ошибками
8. Товары отображаются в админке и на сайте
```

## ⚠️ ВАЖНЫЕ ПРИМЕЧАНИЯ

1. **Весь текстовый контент** будет предоставлен заказчиком отдельно. Верстка выполняется с placeholder'ами.
2. **Иконки** должны быть взяты с референс-сайта или нарисованы в аналогичном стиле.
3. **Цветовая схема** настраивается полностью через админку без перекомпиляции.
4. **SEO** — приоритет №1: SSR, Schema.org, ЧПУ, sitemap, мета-теги.
5. **Безопасность**: админка защищена, API имеет rate limiting.
6. **Масштабируемость**: архитектура позволяет добавлять новые категории и товары без изменения кода.
